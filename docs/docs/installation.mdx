---
title: Installation
description: Install Batchwork and prepare provider access.
---

With uv:

```bash
uv add batchwork-ai
```

With pip:

```bash
pip install batchwork-ai
```

Batchwork requires Python 3.11 or newer. The base install contains HTTPX and Pydantic; provider SDKs are not runtime dependencies.

## Install the CLI

Install the `batchwork` executable as an isolated tool:

```console
uv tool install batchwork-ai
batchwork --version
batchwork --help
```

For an approved one-shot command without a permanent tool install:

```console
uvx --from batchwork-ai batchwork --help
```

Help, version, and shell completion work without configuration, a registry, or credentials. See the [CLI guide](/docs/cli) for human and agent quick starts.

Add the optional Upstash Redis store only when needed:

```bash
uv add "batchwork-ai[redis]"
# or
pip install "batchwork-ai[redis]"
```

Continue with [Configuration](/docs/configuration) for model syntax, provider credentials, endpoint overrides, supported modalities, and batch limits. Credentials may be configured once on `Batchwork`, supplied per call, or read from environment variables.

Submitting a batch performs network requests and creates a real provider job. Provider usage and storage charges may apply.

## Install the Agent Skill

Install the `batchwork-ai` Agent Skill:

```console
npx skills add ajanraj/batchwork-ai@batchwork-ai
```
